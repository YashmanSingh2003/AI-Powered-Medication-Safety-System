# 🏥 AI-Powered Medication Safety System

An AI-driven clinical decision support application that analyzes patient documents, prescriptions, and clinical notes to detect potential medication safety risks.

The system uses large language model (LLM) reasoning to extract structured insights from uploaded medical documents and automatically triggers alert workflows through an automation engine (N8N). This allows healthcare professionals to identify high-risk medication situations and receive automated notifications.

The application is built using Streamlit for the user interface, GPT-based APIs for medical analysis, and N8N for workflow automation.

⚠️ Disclaimer  
This project is designed for research, educational, and demonstration purposes only. It is not intended to replace professional medical advice, diagnosis, or treatment.

---

# 📌 Features

## Clinical Document Analysis
Upload patient reports, prescriptions, or medical notes (PDF format).  
The AI processes unstructured medical text and extracts meaningful clinical information.

## Medication Risk Detection
The AI model evaluates potential medication safety issues including:

- Drug-drug interactions
- Dosage inconsistencies
- Allergy conflicts
- Contraindicated medications
- High-risk prescriptions

## Structured Clinical Insights
The system converts medical text into structured information such as:

- Patient risk level
- Medication warnings
- Key clinical findings
- Recommended safety actions

## Automated Safety Alerts
If high-risk medication patterns are detected:

1. The system triggers an N8N webhook
2. An automated alert workflow runs
3. Email notifications are generated and sent

## Real-Time Monitoring
The platform can be integrated with dashboards to monitor:

- Medication safety trends
- Alert frequency
- Risk classification statistics

# 🔐 Configuration

Create the following file in your project directory:

```
.streamlit/secrets.toml
```

Add your API keys and webhook configuration.

```
OPENROUTER_API_KEY = "PASTE_YOUR_OPENROUTER_KEY_HERE"
N8N_WEBHOOK_URL = "https://your-n8n-host/webhook/your-path"
```

Important:  
Do not commit the secrets.toml file or any API keys to source control.

# 🔗 Testing the N8N Webhook (PowerShell)

Use this command to send a test payload and view the webhook response.

```powershell
$body = @{
    document_text = "Patient prescribed warfarin and ibuprofen"
    question = "Is there a medication safety risk?"
    structured_data = @{ risk_level = "High" }
    recipient_email = "test@example.com"
} | ConvertTo-Json

Invoke-WebRequest -Uri "https://your-n8n-host/webhook/your-path" `
    -Method Post `
    -ContentType "application/json" `
    -Body $body | Select-Object -ExpandProperty Content
```

# 🛠 Troubleshooting

## Webhook returned empty response

The N8N workflow likely does not contain a final HTTP Response node.

Add an HTTP Response node and configure:

Status Code: 200  
Response Body: JSON containing the output fields

---

## KeyError: 'final_answer'

The webhook returned JSON but without the expected keys.

Use the test command above to inspect the raw response and ensure it includes:

- final_answer
- email_body
- email_status

---

## Invalid JSON / Parse Errors

Ensure the N8N HTTP Response node returns valid JSON without HTML wrappers.
